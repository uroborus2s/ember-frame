# 场景图片生成规范

适用项目：《断航故土》。

本规范固定 art-room、prompt-room 和视频生成阶段对场景图片的交付格式。它回答两个问题：每个场景是否都需要近景、中景、远景；每个场景是否都需要多角度、多方向参考图。

## 1. 基本结论

不是每个场景都需要近景、中景和远景，也不是每个场景都需要多角度、多方向图片。

场景图片按用途分层交付：

1. 一次性镜头或蒙太奇空间：只需要该镜头的 9:16 参考帧，甚至可直接 T2V，不强制做可复用场景板。
2. P2 次级可复用场景：至少需要一张 9:16 场景设计板，用多区块展示空间锚点、材质、色彩、入口出口和关键细节。
3. P1 核心复用场景：需要场景设计板，加 3 张生产帧：远景/建立镜头、中景/调度镜头、近景/材质或道具镜头。
4. 复杂动作场景、追逐场景、反复回访场景：在 P1 基础上增加反向角度、侧向通道、入口出口、必要的俯视关系图和关键光照/天气变体。
5. 具体视频镜头：根据 director-room 的 shot-list 生成首帧、尾帧或关键帧；这些帧必须是完整视频画面，不是拼贴设计板。

## 2. 场景图片类型

### location_board

用途：锁定可复用地点的整体视觉身份。

格式：9:16 竖版，建议 1536x2688 或更高；最低 1080x1920。PNG，sRGB。

内容：3 到 6 个可读区块，展示主空间、入口出口、关键道具、材质近看、光照气氛和占领/历史覆盖痕迹。可以像生产设计板，但不能有可读文字标签、水印、UI 或现代排版元素。

使用阶段：P1/P2 场景资产、全局地点参考、后续提示词转换。

不直接作为视频首帧使用，除非裁切后只保留一个完整画面。

### establishing_wide

用途：锁定地点的整体空间关系和第一眼识别。

格式：9:16 生产帧，建议 1440x2560 或 1536x2688；最低 1080x1920。PNG，sRGB。

内容：竖屏里的“远景”不是横向大全景，而是能读出空间层级、入口出口、地貌压迫或生活区域的完整画面。可有人物小剪影用于比例，但不要出现不必要的可识别新角色正脸。

使用阶段：I2V / FLF2V 的地点首帧、转场、场景建立镜头。

### mid_staging

用途：锁定人物行动、站位、路线和镜头轴线。

格式：9:16 生产帧，建议 1440x2560 或 1536x2688；最低 1080x1920。PNG，sRGB。

内容：人物可进入画面的中景调度空间，必须看清门、井、桌、船舷、暗道口、墙体裂缝、追逐方向等调度要素。可放无脸比例人形或背影，除非该图是具体镜头参考帧。

使用阶段：对话、潜入、追捕、护送、过门、过桥、上船、下井等动作。

### detail_closeup

用途：锁定材质、道具和制度痕迹。

格式：优先 9:16 生产帧；材质纹理库可用 1:1，但 1:1 图不得直接作为视频生成首帧。PNG，sRGB。

内容：虫蜡封条、粮牌、旧日月压痕、蜂巢孔、骨钟、蒸汽阀、船工腰绳、磨平炮架接口、烧纸灰、雪泥脚印等可拍细节。

使用阶段：近景插入、REDRAW、局部控制、道具一致性。

### reverse_or_side_angle

用途：防止多镜头剪辑时空间方向混乱。

格式：9:16 生产帧。PNG，sRGB。

内容：同一地点从反向、侧向、入口向出口、出口向入口等视角重拍。必须保持同一套空间锚点，只改变镜头方向。

使用条件：追逐、对峙、多人调度、穿越空间、同一地点跨集回访、观众必须理解入口出口关系时才需要。

### lighting_or_weather_variant

用途：锁定同一地点在剧情状态变化后的视觉连续性。

格式：9:16 生产帧。PNG，sRGB。

内容：同一机位或近似机位下的白天/夜晚、火后/火前、雪前/暴雪、清明白光/火把光、河雾/晴光等状态。

使用条件：地点在剧情中发生可见状态变化时才需要。

### overhead_or_route_map

用途：给导演、分镜和提示词团队理解空间路线，不直接进入视频。

格式：1:1 或 16:9 均可。PNG，sRGB。可以有简化标记，但不要作为最终画面资产。

内容：入口、出口、角色路线、追兵路线、关键遮挡、门窗井洞、上下层关系。

使用条件：旧驿暗道、残阳坳药屋/旧井、寒鸦堡膜廊、渡河蒸汽船、锁喉关、烽燧坍塌等空间方向复杂的场景。

### shot_reference_frame

用途：具体视频镜头的首帧、尾帧或关键帧。

格式：必须是 9:16 完整竖屏视频画面，建议 1440x2560 或 1536x2688；最低 1080x1920。PNG，sRGB。

内容：只表现一个镜头，不做拼贴，不加文字，不加说明标签。人物、道具、光照、方向和景深都应服务该镜头。

使用阶段：I2V、FLF2V、REDRAW、关键动作连续性控制。

## 3. 交付层级

### L0 一次性镜头

适用：黑场、声音转场、短蒙太奇、不可复用过场。

最小交付：`shot_reference_frame` 0 到 1 张。若 generation-plan 标为 T2V 且无稳定角色/地点需求，可不生成场景图。

### L1 次级可复用场景

适用：只出现一集或一段，但后续可能复用的地点。例如灰烬书院、墙下集市、鸣骨岭。

最小交付：1 张 `location_board`。

升级条件：如果该地点承载具体动作调度，追加 1 张 `mid_staging`。

### L2 核心复用场景

适用：多集出现、角色经常回访、地理关系影响剧情的地点。例如残阳坳、旧驿暗道、清明院外署、锁喉关、月下盟书旧驿。

最小交付：

1. `location_board`
2. `establishing_wide`
3. `mid_staging`
4. `detail_closeup`

### L3 复杂动作/空间方向场景

适用：追逐、潜入、坍塌、渡船、堡垒、山口、暗道、需要多镜头剪辑的场景。

最小交付：

1. L2 全部内容
2. `reverse_or_side_angle`
3. `overhead_or_route_map`
4. 必要的 `lighting_or_weather_variant`
5. 对应 shot-list 的 `shot_reference_frame`

## 4. 是否需要近景、中景、远景

近景、中景、远景是镜头语言，不是所有场景资产的硬性清单。

1. P2 场景设计板可以在一张图里同时包含远、中、近的局部区块，不必拆成三张。
2. P1 核心场景必须拆出远景、中景、近景三类生产帧，因为后续镜头会多次引用。
3. L3 场景必须根据动作路线拆角度，远中近只是基础，还要有入口出口、反向和关键状态图。
4. 若场景只用于一个道具特写或一句台词背景，不需要远景。

## 5. 是否需要多角度、多方向

多角度不是为了好看，而是为了防止视频生成时空间漂移。

必须做多角度的情况：

1. 镜头会从入口移动到出口。
2. 角色会转身、回头、追逐、对峙或穿越空间。
3. 同一地点跨多个镜头、多个场次或多个集数出现。
4. 场景内有必须保持位置关系的锚点，例如旧井与药屋、渡船甲板与机械舱、暗道岔口与石门、清明院走廊与蜂巢孔。
5. director-room 标记为 I2V、FLF2V 或 REDRAW 且需要空间连续。

不需要多角度的情况：

1. 一次性氛围镜头。
2. 道具或材质特写。
3. 黑场、声音转场或抽象蒙太奇。
4. 只作为整体视觉风格板的地点。

## 6. 文件命名

全局地点资产：

```text
assets/locations/v01/{loc_id}_board.png
assets/locations/v01/{loc_id}_wide_main.png
assets/locations/v01/{loc_id}_mid_entry.png
assets/locations/v01/{loc_id}_close_detail.png
assets/locations/v01/{loc_id}_reverse.png
assets/locations/v01/{loc_id}_lighting_{state}.png
assets/locations/v01/{loc_id}_route_map.png
```

集级镜头参考帧：

```text
{episode-id}/assets/reference-frames/{shot_id}_first_v01.png
{episode-id}/assets/reference-frames/{shot_id}_last_v01.png
{episode-id}/assets/reference-frames/{shot_id}_key_{beat}_v01.png
```

命名规则：

1. `loc_id` 必须来自 `bible/location-visual-locks.json`。
2. `shot_id` 必须来自 `{episode-id}/shots/shot-list.json`。
3. 不覆盖旧版本。重大方向修订使用 `v02`、`v03`。
4. 失败或漂移图放入 `assets/rejected/`，不得继续作为正式参考。

## 7. 画面禁忌

1. 场景板和生产帧都不要出现可读文字标签、水印、UI、现代图标或说明箭头。
2. 场景图不要为展示宏大而强行加入大规模群演。竖屏短剧优先少量人物、局部动作和清晰锚点。
3. 场景参考图不要让随机角色正脸变成新角色资产。需要比例时用背影、剪影、手部或模糊群像。
4. 同一场景的多角度图必须共享空间锚点、材质、色彩和占领痕迹，不能像不同地点。
5. 用于视频首帧的图必须是单一完整画面，不能是多区拼贴。

## 8. 《断航故土》优先级建议

L2 核心复用场景：残阳坳、旧驿暗道、清明院外署、锁喉关、月下盟书旧驿。

L3 复杂动作场景：残阳坳药屋/旧井、旧驿暗道、金河渡船、寒鸦堡、锁喉关、坍塌烽燧。

L1 次级可复用场景：金河粮磨区、灰烬书院、墙下集市、鸣骨岭、麦浪丘陵农村、北国草原秃山、肃明中枢统治中心。

P2 场景生成当前可先交付 `location_board`；进入具体集数生产时，再由 shot-list 决定是否追加 `mid_staging`、反向角度和首尾帧。
