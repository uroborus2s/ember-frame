# 本地环境与 ComfyUI 基础

本阶段的目标不是让你“看懂大概”，而是让零基础读者能照着完成第一次本地 ComfyUI 操作：打开页面、认出按钮、知道图片从哪里来、知道提示词怎么写、知道模型文件应该放到哪里。

零基础读者先做这份入口文档：[第 0 实操入口：从零创建第一个 ComfyUI + Wan2.2 项目](chapter-00-zero-to-first-project.md)。它会暴露安装、模型、目录和工作流问题；遇到卡点时，再回到第 1-5 章补对应知识。

## 第一部分怎么读

推荐路径是先做入口实操，再补第 1-5 章。这样读者先知道自己会卡在哪里，再带着问题补理论和安装细节。

| 章节 | 你要解决的问题 | 读完后必须会做的动作 |
| --- | --- | --- |
| 第 0 实操入口 | 我现在到底要点哪里、下载什么、放哪里、如何建项目？ | 按 10 步完成第一个 `wan22-first-project`。 |
| 第 1 章 | 我输入的文字、图片和参数最后怎么变成视频？ | 能把一个创意拆成提示词、参考图、参数和输出记录。 |
| 第 2 章 | ComfyUI 怎么打开？本地地址和文件夹在哪里？ | 能打开 `http://127.0.0.1:8000/`，找到 `input/`、`output/`、`models/`。 |
| 第 3 章 | 软件界面里每个区域是什么？ | 能在截图上认出工作流标签、画布、提示词节点、采样器、运行按钮、错误提示。 |
| 第 4 章 | 模型文件从哪里来，放哪里，为什么缺模型会报错？ | 能按文件类型把 Wan2.2 模型放进 `diffusion_models/`、`text_encoders/`、`vae/`。 |
| 第 5 章 | 第一次运行到底填什么？ | 能拖入 JSON，放入参考图，改提示词，设置宽高/帧数/seed，点击运行并记录结果。 |

## 图片、截图和输入从哪里来

教程里会严格区分三类图片：

| 类型 | 例子 | 来源 | 你应该怎么用 |
| --- | --- | --- | --- |
| 软件截图 | ComfyUI 首页、模型目录、错误提示 | 本机实际打开 ComfyUI 或本机文件夹后截图 | 用来找按钮、找目录、对照你自己的界面。 |
| 模板预览图 | Wan2.2 模板自带预览 | 本机 ComfyUI 模板包或课程资产 | 用来理解模板用途，不等于你已经生成成功。 |
| 练习输入图 | 第 5 章 starter image | 课程提供的入门参考图 | 先用它验证 `LoadImage` 和工作流链路，之后换成你自己的素材。 |

跟操时你要填写的输入来自这里：

| 输入项 | 从哪里获得 | 第一次怎么填 |
| --- | --- | --- |
| 参考图 | 课程提供的 `docs/assets/inputs/chapter-05/05-input-starter-product.png`，或你自己的清晰图片 | 先用课程图，确认会加载图片后再换自己的图。 |
| 正向提示词 | 按第 1 章公式写：主体 + 动作 + 场景 + 镜头 + 风格 | 第一次直接抄第 5 章给出的短提示词。 |
| 负向提示词 | 模板默认给出，用来减少字幕、水印、模糊、畸形等问题 | 第一次不要改，等跑通后再微调。 |
| 模型文件名 | 工作流 JSON 和第 4 章模型清单 | 保留官方文件名，不要改中文名。 |
| 宽高/帧数 | 第 5 章显存表 | 8GB/12GB 先降到短帧数和低分辨率。 |
| seed/steps/cfg | 工作流节点里的采样参数 | 第一次固定 seed，只改一次一个参数。 |

## 截图怎么自己保存

每章都要求你保存截图，不是为了好看，而是为了排错。你至少要保存：

1. 当前 ComfyUI 页面全景，能看到工作流和运行按钮。
2. 参数节点局部，能看到你改过的提示词、宽高、帧数、seed。
3. 错误提示或输出目录，能证明这次运行的结果。

命名规则：

```text
章节号-步骤号-内容.png
例：05-03-first-run-input-callouts.png
例：05-04-wan22-5b-input-map.svg
```
