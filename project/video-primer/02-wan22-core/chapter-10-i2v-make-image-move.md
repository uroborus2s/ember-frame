# 第 10 章：I2V 图生视频：让一张图动起来

> 建议时长：75-90 分钟
> 适用平台：macOS / Windows / Linux
> 本章目标：让学习者用一张参考图控制主体外观，并用文字控制运动。

| 主产出 | 2 段 I2V 练习记录和参考图检查表 |

## 实操验证边界

本章随仓库提供工作流、界面截图和记录表。生成结果、耗时、显存峰值和质量评分必须由学习者在自己的 ComfyUI 环境中记录；凡未完成实测的位置，一律标为 `待实测`，不得写成已生成。

这不是跳过实操，而是把可验证和不可验证分开：界面、模板、参数、目录、日志可以实测；真正的视频质量只能在模型文件到位后验证。

## 显存档位建议

| 显存 | 推荐做法 | 风险控制 |
| ---: | --- | --- |
| 8GB | 只做低分辨率、短帧数、单 seed；优先 5B 或只完成界面和参数演练。 | 不要同时加载 14B high/low 两个大模型；失败时先降分辨率和帧数。 |
| 12GB | 可以做 5B 完整练习，14B 只做小尺寸验证或使用 fp8/量化版本。 | 每次只跑一个候选，运行前关闭其他占显存软件。 |
| 16GB | 可以系统练习 14B T2V/I2V 的小中尺寸流程，保留草稿参数。 | 先用短帧数筛 seed，再放大，不要一开始追求 720P 长视频。 |
| 24GB | 可以完成本章 I2V 的标准练习，并做 2-4 个候选对比。 | 仍然要记录 seed、模型、steps、分辨率、帧数和耗时。 |

## 跟做实操

1. 打开 ComfyUI 首页。
2. 优先把本章提供的 JSON 拖到 ComfyUI 画布；如果要用软件内模板入口，请打开当前界面的“浏览模板”，搜索 `Wan2.2` 并选择对应视频模板。
3. 按截图定位提示词、模型、尺寸、帧数、seed 和输出节点。
4. 如果节点提示缺模型，先记录缺失文件名，不要乱改节点。
5. 按显存档位选择草稿参数。
6. 运行后把输出文件名、seed、参数、截图写入记录表。

## 知识点 1：参考图质量

参考图越清晰、主体越明确，I2V 越容易保持外观。模糊图和复杂背景会放大漂移。

### 实例 A：清晰人像生成微表情

| 项目 | 内容 |
| --- | --- |
| 输入 | 正面半身清晰人像。 |
| 操作 | 上传图像，提示词写眨眼、轻微微笑。 |
| 预期现象 | 人物外观预期尽量保持，动作小而稳定。 |
| 判断原则 | 人物 I2V 先从小动作开始。 |

操作流程：

1. 把图放入 input。
2. 在 Load Image 选择图片。
3. 填写微动作提示词。
4. 记录面部稳定性。


### 实例 B：产品图生成慢速旋转

| 项目 | 内容 |
| --- | --- |
| 输入 | 背景干净的产品图。 |
| 操作 | 提示词写 slow rotation 和 studio light sweep。 |
| 预期现象 | 产品应保持形状并有轻微运动。 |
| 判断原则 | 产品图比复杂人物动作更适合初学者。 |

操作流程：

1. 准备产品图。
2. 选择 I2V 模板。
3. 输入产品运动。
4. 检查产品轮廓。


## 知识点 2：动作提示词

I2V 文字负责动作，不要让文字推翻参考图外观。动作越大，越容易变形。

### 实例 A：人物转头

| 项目 | 内容 |
| --- | --- |
| 输入 | `the person gently turns head and smiles`。 |
| 操作 | 只写小幅动作。 |
| 预期现象 | 动作自然，身份更容易保持。 |
| 判断原则 | 转头比奔跑更适合第一轮。 |

操作流程：

1. 保持参考图。
2. 输入小动作。
3. 运行一版。
4. 检查脸部漂移。


### 实例 B：产品悬浮光扫

| 项目 | 内容 |
| --- | --- |
| 输入 | `the product floats slightly, light sweeps across the surface`。 |
| 操作 | 让运动主要来自光线和镜头。 |
| 预期现象 | 产品结构更稳定。 |
| 判断原则 | 产品镜头优先让光动，而不是让产品大幅变形。 |

操作流程：

1. 保留产品居中。
2. 输入光线运动。
3. 运行或记录。
4. 检查品牌/轮廓稳定。


## 知识点 3：构图保持

I2V 不等于锁死画面。构图要靠参考图、提示词和尺寸共同约束。

### 实例 A：半身人物构图

| 项目 | 内容 |
| --- | --- |
| 输入 | 半身人像，提示词加入 `medium close-up, centered composition`。 |
| 操作 | 保持同宽高比例。 |
| 预期现象 | 人物仍在画面中心。 |
| 判断原则 | 尺寸比例变化太大会破坏构图。 |

操作流程：

1. 记录原图比例。
2. 设置相近宽高。
3. 加入构图词。
4. 观察主体位置。


### 实例 B：产品居中构图

| 项目 | 内容 |
| --- | --- |
| 输入 | 产品图，提示词加入 `centered product shot`。 |
| 操作 | 不加入复杂场景。 |
| 预期现象 | 产品居中，背景轻微变化。 |
| 判断原则 | 产品展示优先保持主体，不追求复杂背景。 |

操作流程：

1. 选择干净产品图。
2. 保持居中描述。
3. 运行对比。
4. 标注偏移程度。

## 常见错误与排查

| 错误 | 常见原因 | 处理 |
| --- | --- | --- |
| 节点红框提示缺模型 | 模型文件没有放到工作流要求的目录。 | 先看本章模型清单，把文件放入 `models/diffusion_models/`、`models/text_encoders/`、`models/vae/` 或 `models/loras/`。 |
| 显存不足或运行中断 | 分辨率、帧数、steps 或模型规模超过本机显存。 | 按显存表降到短帧数、低分辨率、单 seed，再逐步放大。 |
| 结果无法复现 | 没有记录 seed、模型、提示词、工作流版本。 | 每次运行后立刻填写实操记录表。 |
