## 实操验证边界

本章随仓库提供工作流、界面截图和记录表。生成结果、耗时、显存峰值和质量评分必须由学习者在自己的 ComfyUI 环境中记录；凡未完成实测的位置，一律标为 `待实测`，不得写成已生成。

这不是跳过实操，而是把可验证和不可验证分开：界面、模板、参数、目录、日志可以实测；真正的视频质量只能在模型文件到位后验证。

## 显存档位建议

| 显存 | 推荐做法 | 风险控制 |
| ---: | --- | --- |
| 8GB | 只做低分辨率、短帧数、单 seed；优先 5B 或只完成界面和参数演练。 | 不要同时加载 14B high/low 两个大模型；失败时先降分辨率和帧数。 |
| 12GB | 可以做 5B 完整练习，14B 只做小尺寸验证或使用 fp8/量化版本。 | 每次只跑一个候选，运行前关闭其他占显存软件。 |
| 16GB | 可以系统练习 14B T2V/I2V 的小中尺寸流程，保留草稿参数。 | 先用短帧数筛 seed，再放大，不要一开始追求 720P 长视频。 |
| 24GB | 可以完成本章 T2V 的标准练习，并做 2-4 个候选对比。 | 仍然要记录 seed、模型、steps、分辨率、帧数和耗时。 |

## 跟做实操

1. 打开 ComfyUI 首页。
2. 优先把本章提供的 JSON 拖到 ComfyUI 画布；如果要用软件内模板入口，请打开当前界面的“浏览模板”，搜索 `Wan2.2` 并选择对应视频模板。
3. 按截图定位提示词、模型、尺寸、帧数、seed 和输出节点。
4. 如果节点提示缺模型，先记录缺失文件名，不要乱改节点。
5. 按显存档位选择草稿参数。
6. 运行后把输出文件名、seed、参数、截图写入记录表。

## 知识点 1：提示词基础结构

T2V 提示词至少包含主体、动作、场景。缺一个，模型就会自己补，结果更难控。

### 实例 A：人物 + 动作 + 场景

| 项目 | 内容 |
| --- | --- |
| 输入 | `a young detective walks slowly through a rainy neon street at night`。 |
| 操作 | 填入正向提示词，负向提示词只写常见缺陷。 |
| 预期现象 | 在条件满足时预期生成雨夜街道人物行走镜头。 |
| 判断原则 | 主体是 detective，动作是 walks slowly，场景是 rainy neon street。 |

操作流程：

1. 打开 T2V 模板。
2. 找到正向提示词框。
3. 输入英文提示词。
4. 固定 seed 运行或记录缺模型。


### 实例 B：物体 + 运动 + 环境

| 项目 | 内容 |
| --- | --- |
| 输入 | `a silver headphone floating and slowly rotating in a clean studio with soft rim light`。 |
| 操作 | 同样参数只替换提示词。 |
| 预期现象 | 在条件满足时预期生成产品展示镜头。 |
| 判断原则 | 这是产品镜头，不要加入人物和复杂背景。 |

操作流程：

1. 复制实例 A 参数。
2. 替换主体和动作。
3. 保持 seed 或记录新 seed。
4. 比较主体稳定性。


## 知识点 2：镜头描述

镜头描述告诉模型“摄影机怎么拍”。固定镜头更稳，推镜、摇镜会增加运动风险。

### 实例 A：固定镜头雨夜街道

| 项目 | 内容 |
| --- | --- |
| 输入 | 在提示词末尾加入 `static camera, medium shot`。 |
| 操作 | 只添加固定镜头描述。 |
| 预期现象 | 运动主要来自人物，背景更稳定。 |
| 判断原则 | 固定镜头适合第一次测试。 |

操作流程：

1. 保留主体和场景。
2. 加入 static camera。
3. 运行一版。
4. 观察背景是否乱动。


### 实例 B：缓慢推镜未来城市

| 项目 | 内容 |
| --- | --- |
| 输入 | 加入 `slow dolly in, wide shot, futuristic city skyline`。 |
| 操作 | 只改变镜头运动和景别。 |
| 预期现象 | 画面预期有推进感，但更容易变形。 |
| 判断原则 | 镜头运动越复杂，越要降低动作复杂度。 |

操作流程：

1. 保留同一 seed。
2. 替换镜头描述。
3. 运行或记录风险。
4. 对比运动幅度。


## 知识点 3：风格描述

风格词影响光线、色彩和质感。风格不要堆太多，否则主体和动作会被稀释。

### 实例 A：纪录片自然光

| 项目 | 内容 |
| --- | --- |
| 输入 | `documentary style, natural light, realistic color`。 |
| 操作 | 加入风格词，不改主体。 |
| 预期现象 | 画面预期更自然。 |
| 判断原则 | 自然光适合人物和生活类镜头。 |

操作流程：

1. 复制基础提示词。
2. 添加纪录片风格。
3. 保持参数。
4. 记录画面真实感。


### 实例 B：电影预告片高反差

| 项目 | 内容 |
| --- | --- |
| 输入 | `cinematic trailer, high contrast lighting, dramatic shadows`。 |
| 操作 | 只改风格词。 |
| 预期现象 | 画面对比更强，氛围更戏剧化。 |
| 判断原则 | 这是风格变量实验。 |

操作流程：

1. 保留主体动作。
2. 替换风格短语。
3. 运行或记录预期。
4. 对比光线和色彩。

## 常见错误与排查

| 错误 | 常见原因 | 处理 |
| --- | --- | --- |
| 节点红框提示缺模型 | 模型文件没有放到工作流要求的目录。 | 先看本章模型清单，把文件放入 `models/diffusion_models/`、`models/text_encoders/`、`models/vae/` 或 `models/loras/`。 |
| 显存不足或运行中断 | 分辨率、帧数、steps 或模型规模超过本机显存。 | 按显存表降到短帧数、低分辨率、单 seed，再逐步放大。 |
| 结果无法复现 | 没有记录 seed、模型、提示词、工作流版本。 | 每次运行后立刻填写实操记录表。 |
