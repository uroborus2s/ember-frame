# 第 6 章：Wan2.2 模型家族与任务选择

> 建议时长：75-90 分钟
> 适用平台：macOS / Windows / Linux
> 本章目标：让学习者知道什么时候用 5B、14B T2V、14B I2V 和 FLF2V。

## 实操验证边界

本章随仓库提供工作流、界面截图和记录表。生成结果、耗时、显存峰值和质量评分必须由学习者在自己的 ComfyUI 环境中记录；凡未完成实测的位置，一律标为 `待实测`，不得写成已生成。

这不是跳过实操，而是把可验证和不可验证分开：界面、模板、参数、目录、日志可以实测；真正的视频质量只能在模型文件到位后验证。

## 显存档位建议

| 显存 | 推荐做法 | 风险控制 |
| ---: | --- | --- |
| 8GB | 只做低分辨率、短帧数、单 seed；优先 5B 或只完成界面和参数演练。 | 不要同时加载 14B high/low 两个大模型；失败时先降分辨率和帧数。 |
| 12GB | 可以做 5B 完整练习，14B 只做小尺寸验证或使用 fp8/量化版本。 | 每次只跑一个候选，运行前关闭其他占显存软件。 |
| 16GB | 可以系统练习 14B T2V/I2V 的小中尺寸流程，保留草稿参数。 | 先用短帧数筛 seed，再放大，不要一开始追求 720P 长视频。 |
| 24GB | 可以完成本章 Wan2.2 任务选择 的标准练习，并做 2-4 个候选对比。 | 仍然要记录 seed、模型、steps、分辨率、帧数和耗时。 |

## 跟做实操

1. 写下当前任务：只有文字、已有图、还是有首尾帧。
2. 按任务选择表选择模型。
3. 按显存档位选择 5B 草稿或 14B 精修。
4. 检查模型文件是否齐全。
5. 记录选择理由。
6. 只进入一个工作流，不要同时切换多个方向。

## Wan2.2 工作流矩阵

本课程实际使用的是 Wan2.2 官方模型与 Comfy-Org ComfyUI repackaged 权重/工作流。模型文件名和依赖以仓库内置 JSON 及当日官方页面为准；如果网页说明和 JSON 不一致，先按 JSON 排查。

| 工作流 | 输入条件 | diffusion model | VAE | text encoder | LoRA | 适用场景 | 不适用场景 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| TI2V-5B | 文本，可配输入图 | `wan2.2_ti2v_5B_fp16.safetensors` | `wan2.2_vae.safetensors` | `umt5_xxl_fp8_e4m3fn_scaled.safetensors` | 不需要 | 低显存跑通、草稿筛选、第一次项目 | 直接追求最终高质量成片 |
| 14B T2V | 文本 | `wan2.2_t2v_high_noise_14B_fp8_scaled.safetensors` + `wan2.2_t2v_low_noise_14B_fp8_scaled.safetensors` | `wan_2.1_vae.safetensors` | `umt5_xxl_fp8_e4m3fn_scaled.safetensors` | 4-step LoRA 仅加速分组需要 | 文本镜头精修、环境和概念镜头 | 需要严格保持特定人物/产品外观 |
| 14B I2V | 单张参考图 + 文本 | `wan2.2_i2v_high_noise_14B_fp8_scaled.safetensors` + `wan2.2_i2v_low_noise_14B_fp8_scaled.safetensors` | `wan_2.1_vae.safetensors` | `umt5_xxl_fp8_e4m3fn_scaled.safetensors` | 4-step LoRA 仅加速分组需要 | 产品、角色、场景参考图运动化 | 大幅转身、换装、复杂遮挡 |
| 14B FLF2V | 首帧 + 尾帧 + 文本 | 复用 I2V high/low 模型位置 | `wan_2.1_vae.safetensors` | `umt5_xxl_fp8_e4m3fn_scaled.safetensors` | 4-step LoRA 仅加速分组需要 | 明确开始和结束状态的过渡 | 首尾帧语义差太大、构图跨度过大 |

5B 是 `Wan2.2-TI2V-5B` hybrid workflow。课程里说“5B 文本模式”时，意思是在该 TI2V 工作流中主要使用文本条件做草稿，不代表存在另一个独立的 5B T2V 模型。

## 知识点 1：任务类型选择

任务类型由输入决定。没有图就用 T2V，有单张图就用 I2V/TI2V，有明确开始和结束就用 FLF2V。

### 实例 A：只有创意文本时选择 T2V

| 项目 | 内容 |
| --- | --- |
| 输入 | “雨夜街道上，一位侦探慢慢走向霓虹灯”。 |
| 操作 | 选择 14B T2V；低显存先用 5B TI2V 工作流的文本条件做草稿。 |
| 预期现象 | 在模型文件完整、显存足够、工作流无红节点时，得到从文字直接生成的镜头。 |
| 判断原则 | 没有参考图时，不要用 I2V 假装控制角色。 |

操作流程：

1. 判断输入只有文字。
2. 选择 T2V。
3. 填写主体、动作、场景。
4. 记录模型选择理由。


### 实例 B：已有产品图时选择 I2V/TI2V

| 项目 | 内容 |
| --- | --- |
| 输入 | 一张耳机产品图。 |
| 操作 | 选择 I2V 或 TI2V，让产品参考图参与外观控制，并用文字描述光线运动。 |
| 预期现象 | 产品外观比纯文本更稳定，但仍可能被模型软性改写。 |
| 判断原则 | 参考图主要提供外观条件，文字主要提供动作和氛围意图，二者都不是硬约束。 |

操作流程：

1. 准备清晰产品图。
2. 加载 I2V/TI2V。
3. 输入产品动作提示词。
4. 观察是否保持外观。


## 知识点 2：5B 与 14B 的取舍

5B 适合快速跑通和低显存学习；14B 适合更高质量，但模型、显存和时间成本更高。

### 实例 A：5B 快速预览

| 项目 | 内容 |
| --- | --- |
| 输入 | 短提示词和低分辨率。 |
| 操作 | 用 5B 测试构图和运动方向。 |
| 预期现象 | 快速得到方向判断。 |
| 判断原则 | 5B 是筛选工具，不一定是最终成片工具。 |

操作流程：

1. 选择 5B 模板。
2. 设置短帧数。
3. 跑 1 个 seed。
4. 记录是否值得精修。


### 实例 B：14B 精修候选镜头

| 项目 | 内容 |
| --- | --- |
| 输入 | 从 5B 筛出的提示词和 seed。 |
| 操作 | 切换 14B 对候选进行高质量重跑。 |
| 预期现象 | 画面细节和语义可能更好，但耗时和显存占用会上升。 |
| 判断原则 | 14B 应用于候选，不用于无控制乱试。 |

操作流程：

1. 选择候选提示词。
2. 加载 14B 工作流。
3. 保留记录表。
4. 对比 5B 结果。


## 知识点 3：高噪声/低噪声阶段

14B 工作流常把生成拆成 high noise 和 low noise 阶段。高噪声决定大结构，低噪声负责细化。

### 实例 A：14B T2V 双模型

| 项目 | 内容 |
| --- | --- |
| 输入 | T2V high noise 和 low noise 两个模型。 |
| 操作 | 两个都放入 diffusion_models。 |
| 预期现象 | 工作流可完整加载。 |
| 判断原则 | 只放一个模型会导致流程不完整。 |

操作流程：

1. 下载 high noise。
2. 下载 low noise。
3. 放同一目录。
4. 在节点中分别选择。


### 实例 B：14B I2V/FLF2V 双模型

| 项目 | 内容 |
| --- | --- |
| 输入 | I2V high noise 和 low noise。 |
| 操作 | I2V 与 FLF2V 共用这组模型位置。 |
| 预期现象 | 单图动起来或首尾帧过渡。 |
| 判断原则 | 在 ComfyUI 这套模板中，FLF2V 使用 I2V high/low 模型位置；这是工作流层面的复用，不要扩展成所有发布形态的模型分类结论。 |

操作流程：

1. 按 I2V 清单放模型。
2. 加载 FLF2V。
3. 选择首尾帧。
4. 记录模型组合。

## 常见错误与排查

| 错误 | 常见原因 | 处理 |
| --- | --- | --- |
| 节点红框提示缺模型 | 模型文件没有放到工作流要求的目录。 | 先看本章模型清单，把文件放入 `models/diffusion_models/`、`models/text_encoders/`、`models/vae/` 或 `models/loras/`。 |
| 显存不足或运行中断 | 分辨率、帧数、steps 或模型规模超过本机显存。 | 按显存表降到短帧数、低分辨率、单 seed，再逐步放大。 |
| 结果无法复现 | 没有记录 seed、模型、提示词、工作流版本。 | 每次运行后立刻填写实操记录表。 |
