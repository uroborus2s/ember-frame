## 实操验证边界

本章随仓库提供工作流、界面截图和记录表。生成结果、耗时、显存峰值和质量评分必须由学习者在自己的 ComfyUI 环境中记录；凡未完成实测的位置，一律标为 `待实测`，不得写成已生成。

这不是跳过实操，而是把可验证和不可验证分开：界面、模板、参数、目录、日志可以实测；真正的视频质量只能在模型文件到位后验证。

## 显存档位建议

| 显存 | 推荐做法 | 风险控制 |
| ---: | --- | --- |
| 8GB | 只做低分辨率、短帧数、单 seed；优先 5B 或只完成界面和参数演练。 | 不要同时加载 14B high/low 两个大模型；失败时先降分辨率和帧数。 |

## 跟做实操

1. 打开 ComfyUI 首页。
2. 优先把本章提供的 JSON 拖到 ComfyUI 画布；如果要用软件内模板入口，请打开当前界面的“浏览模板”，搜索 `Wan2.2` 并选择对应视频模板。
3. 按截图定位提示词、模型、尺寸、帧数、seed 和输出节点。
4. 如果节点提示缺模型，先记录缺失文件名，不要乱改节点。
5. 按显存档位选择草稿参数。
6. 运行后把输出文件名、seed、参数、截图写入记录表。

## 知识点 1：提示词字段化

字段化让提示词可修改、可比较。不要把所有想法塞成一句长句。

### 实例 A：人物广告镜头模板

| 项目 | 内容 |
| --- | --- |
| 输入 | 主体：年轻女性；动作：拿起耳机；镜头：近景推近；光线：柔和轮廓光。 |
| 操作 | 按字段拼成一条英文提示词。 |
| 预期现象 | 得到结构清晰的广告镜头提示词。 |
| 判断原则 | 字段越清楚，后续越好改变量。 |

操作流程：

1. 写出主体字段。
2. 写动作字段。
3. 写镜头和光线字段。
4. 合并成一句提示词。


### 实例 B：风景空镜模板

| 项目 | 内容 |
| --- | --- |
| 输入 | 主体：雪山；动作：云雾流动；镜头：广角固定；风格：纪录片自然色。 |
| 操作 | 沿用相同字段，不写人物。 |
| 预期现象 | 得到风景空镜提示词。 |
| 判断原则 | 不同题材也用同一字段框架。 |

操作流程：

1. 删除人物字段。
2. 强化环境和天气。
3. 保持镜头字段。
4. 记录版本号。


## 知识点 2：负向提示词

负向提示词用于减少常见错误，不是万能修复。不要把正向目标写进负向。

### 实例 A：减少肢体错误

| 项目 | 内容 |
| --- | --- |
| 输入 | `deformed hands, extra fingers, distorted face, broken anatomy`。 |
| 操作 | 放入负向提示词框。 |
| 预期现象 | 人物错误可能减少，但不会完全消失。 |
| 判断原则 | 负向只能降低概率，不能代替好参考图和合理动作。 |

操作流程：

1. 找到负向提示词框。
2. 输入常见缺陷。
3. 保持正向不变。
4. 对比失败帧。


### 实例 B：减少错误文字和标志

| 项目 | 内容 |
| --- | --- |
| 输入 | `wrong text, unreadable logo, watermark, artifacts`。 |
| 操作 | 用于产品或广告画面。 |
| 预期现象 | 错误文字可能减少。 |
| 判断原则 | 如果需要准确文字，后期叠字通常更可靠。 |

操作流程：

1. 在产品提示词中加入负向。
2. 不要要求模型生成精确商标。
3. 输出后检查文字。
4. 需要准确文字时留给后期。


## 知识点 3：迭代修改

一次只改一个变量。否则你不知道结果变化来自动作、镜头、光线还是风格。

### 实例 A：只改动作词

| 项目 | 内容 |
| --- | --- |
| 输入 | `walks slowly` 改为 `turns head gently`。 |
| 操作 | 其他字段不变。 |
| 预期现象 | 主体从行走变为转头。 |
| 判断原则 | 这轮测试动作变量。 |

操作流程：

1. 复制 V1 提示词。
2. 只替换动作字段。
3. 保留 seed。
4. 记录运动变化。


### 实例 B：只改光线词

| 项目 | 内容 |
| --- | --- |
| 输入 | `natural light` 改为 `hard backlight`。 |
| 操作 | 动作、主体、镜头不变。 |
| 预期现象 | 画面氛围变强，主体轮廓变化。 |
| 判断原则 | 这轮测试光线变量。 |

操作流程：

1. 复制 V1。
2. 只替换光线字段。
3. 运行对比。
4. 写出光线变化是否可用。

## 常见错误与排查

| 错误 | 常见原因 | 处理 |
| --- | --- | --- |
| 节点红框提示缺模型 | 模型文件没有放到工作流要求的目录。 | 先看本章模型清单，把文件放入 `models/diffusion_models/`、`models/text_encoders/`、`models/vae/` 或 `models/loras/`。 |
| 显存不足或运行中断 | 分辨率、帧数、steps 或模型规模超过本机显存。 | 按显存表降到短帧数、低分辨率、单 seed，再逐步放大。 |
| 结果无法复现 | 没有记录 seed、模型、提示词、工作流版本。 | 每次运行后立刻填写实操记录表。 |
