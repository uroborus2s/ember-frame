# 第 9 章：采样参数、种子、步数与分辨率

> 建议时长：75-90 分钟
> 适用平台：macOS / Windows / Linux
> 本章目标：让学习者知道 seed、steps、分辨率和帧数如何影响速度、质量和复现。

## 实操验证边界

本章随仓库提供工作流、界面截图和记录表。生成结果、耗时、显存峰值和质量评分必须由学习者在自己的 ComfyUI 环境中记录；凡未完成实测的位置，一律标为 `待实测`，不得写成已生成。

这不是跳过实操，而是把可验证和不可验证分开：界面、模板、参数、目录、日志可以实测；真正的视频质量只能在模型文件到位后验证。

## 显存档位建议

| 显存 | 推荐做法 | 风险控制 |
| ---: | --- | --- |
| 8GB | 只做低分辨率、短帧数、单 seed；优先 5B 或只完成界面和参数演练。 | 不要同时加载 14B high/low 两个大模型；失败时先降分辨率和帧数。 |
| 12GB | 可以做 5B 完整练习，14B 只做小尺寸验证或使用 fp8/量化版本。 | 每次只跑一个候选，运行前关闭其他占显存软件。 |
| 16GB | 可以系统练习 14B T2V/I2V 的小中尺寸流程，保留草稿参数。 | 先用短帧数筛 seed，再放大，不要一开始追求 720P 长视频。 |
| 24GB | 可以完成本章 采样参数 的标准练习，并做 2-4 个候选对比。 | 仍然要记录 seed、模型、steps、分辨率、帧数和耗时。 |

## 跟做实操

1. 打开 ComfyUI 首页。
2. 优先把本章提供的 JSON 拖到 ComfyUI 画布；如果要用软件内模板入口，请打开当前界面的“浏览模板”，搜索 `Wan2.2` 并选择对应视频模板。
3. 按截图定位提示词、模型、尺寸、帧数、seed 和输出节点。
4. 如果节点提示缺模型，先记录缺失文件名，不要乱改节点。
5. 按显存档位选择草稿参数。
6. 运行后把输出文件名、seed、参数、截图写入记录表。

## 知识点 1：seed 与可复现

seed 是随机起点。固定 seed 才能判断“只改一个变量”的影响，但它不是跨环境的绝对复现承诺。模型文件、工作流图、节点版本、采样器、scheduler、精度、后端和随机化模式基本一致时，seed 比较才有意义；跨 GPU 后端或跨版本不保证逐像素一致。

### 实例 A：固定 seed 只改提示词

| 项目 | 内容 |
| --- | --- |
| 输入 | seed=1234，两条提示词。 |
| 操作 | 保留所有参数，只替换动作词。 |
| 预期现象 | 差异主要来自提示词。 |
| 判断原则 | 这是提示词实验，不是随机实验。 |

操作流程：

1. 设置固定 seed，并关闭会自动随机 seed 的选项。
2. 运行提示词 A。
3. 复制并改提示词。
4. 记录差异。


### 实例 B：固定提示词只改 seed

| 项目 | 内容 |
| --- | --- |
| 输入 | 同一提示词，seed=1234 和 seed=5678。 |
| 操作 | 只改 seed。 |
| 预期现象 | 构图和细节会变化。 |
| 判断原则 | 这是随机性筛选。 |

操作流程：

1. 保留提示词。
2. 改 seed。
3. 运行 B。
4. 选更稳定的一版。


## 知识点 2：steps 与质量/耗时

steps 增加通常会提升细化时间，但不是越高越好。不同 Wan2.2 工作流不能套用同一组 steps 经验：5B TI2V、14B high/low noise 双阶段、4-step LoRA 加速分组的采样设定不同，必须先按当前模板默认值建立基线，再做单变量实验。

### 实例 A：低步数草稿

| 项目 | 内容 |
| --- | --- |
| 输入 | 使用当前工作流的低成本草稿 steps，或先按模板默认值减少一档。 |
| 操作 | 快速跑通和筛方向。 |
| 预期现象 | 速度快，细节可能粗。 |
| 判断原则 | 低步数用于筛选，不用于最终交付。 |

操作流程：

1. 设置低 steps。
2. 运行单 seed。
3. 记录耗时。
4. 标注草稿。


### 实例 B：高步数精修

| 项目 | 内容 |
| --- | --- |
| 输入 | 只在同一工作流内，把候选镜头从草稿 steps 调回模板默认或略高一档。 |
| 操作 | 只对候选镜头增加 steps。 |
| 预期现象 | 耗时增加，细节可能改善。 |
| 判断原则 | 只对值得保留的 seed 精修。 |

操作流程：

1. 选择候选 seed。
2. 提高 steps。
3. 其他不变。
4. 比较细节和运动。


## 知识点 3：分辨率与帧数

分辨率影响画面尺寸，帧数影响时长和运动。两者都会增加资源压力。

### 实例 A：低分辨率短帧数

| 项目 | 内容 |
| --- | --- |
| 输入 | 640x640，49 帧。 |
| 操作 | 低成本预览。 |
| 预期现象 | 可快速判断方向。 |
| 判断原则 | 这是低显存默认起点。 |

操作流程：

1. 设置宽高。
2. 设置帧数。
3. 运行或记录风险。
4. 写入表格。


### 实例 B：高分辨率长帧数

| 项目 | 内容 |
| --- | --- |
| 输入 | 接近 720P，81-121 帧。 |
| 操作 | 只在 16GB/24GB 或确认可运行时尝试。 |
| 预期现象 | 耗时和显存明显增加。 |
| 判断原则 | 最终输出要建立在草稿通过之后。 |

操作流程：

1. 复制草稿参数。
2. 逐步提高分辨率。
3. 再提高帧数。
4. 任一步失败就回退。

## 常见错误与排查

| 错误 | 常见原因 | 处理 |
| --- | --- | --- |
| 节点红框提示缺模型 | 模型文件没有放到工作流要求的目录。 | 先看本章模型清单，把文件放入 `models/diffusion_models/`、`models/text_encoders/`、`models/vae/` 或 `models/loras/`。 |
| 显存不足或运行中断 | 分辨率、帧数、steps 或模型规模超过本机显存。 | 按显存表降到短帧数、低分辨率、单 seed，再逐步放大。 |
| 结果无法复现 | 没有记录 seed、模型、提示词、工作流版本。 | 每次运行后立刻填写实操记录表。 |
