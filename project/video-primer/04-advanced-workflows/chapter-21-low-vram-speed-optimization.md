## 显存档位建议

| 显存 | 推荐做法 | 风险控制 |
| ---: | --- | --- |
| 8GB | 只做 低显存优化 的低分辨率草稿：优先 5B、短帧数、单 seed。 | 不同时跑多个候选，不加载 14B 双阶段大模型。 |
| 12GB | 可完成 5B 完整练习；14B 只做小尺寸验证或等待 fp8/量化版本。 | 每次只改一个变量，失败先降帧数。 |
| 16GB | 可做 14B 小中尺寸流程，并保留草稿和精修两套参数。 | 先筛 seed，再提升分辨率。 |
| 24GB | 可做 低显存优化 的标准练习和 2-4 个候选对比。 | 仍要记录 seed、模型、steps、宽高、帧数、耗时。 |

## 知识点：降低显存占用

先降宽高和帧数，再考虑模型规模。

### 实例：降低分辨率帧数

| 项目 | 内容 |
| --- | --- |
| 输入 | 把 1280x704、121 帧降到 640x640、49 帧。 |
| 操作 | 只测试变量 `降级=尺寸帧数`，其他参数保持不变。 |
| 预期现象 | 显存压力和等待时间下降，画面只用于草稿判断。 |
| 判断原则 | 低显存先降尺寸和帧数。 |

操作流程：

1. 打开本章对应的 Wan2.2 工作流或项目记录表。
2. 在记录表里写下变量：`降级=尺寸帧数`。
3. 输入本例提示词、素材或镜头要求。
4. 按显存档位选择草稿参数；本机缺模型时记录缺失文件，不伪造输出。
5. 截图保存节点、参数或项目表，并写下本例是否可进入下一步。

## 知识点：速度优化

速度优化不是一味降质量，而是先筛选再精修。

### 实例：草稿筛 seed

| 项目 | 内容 |
| --- | --- |
| 输入 | 固定提示词，连续测试 seed=101、102、103。 |
| 操作 | 只测试变量 `速度=草稿`，其他参数保持不变。 |
| 预期现象 | 若生成成功，记录 3 个构图候选并选择最稳定的一版；失败也要记录错误原文。 |
| 判断原则 | 先筛 seed 再精修。 |

## 常见错误与排查

| 错误 | 常见原因 | 处理 |
| --- | --- | --- |
| 只写概念没有变量 | 不知道本章到底要验证什么。 | 每个实例只设置一个核心变量，并写入记录表。 |
| 结果失败但没有记录 | 缺少 seed、模型、提示词或截图。 | 先补记录，再决定是否重跑。 |
| 本机缺模型却写“已生成” | 伪造输出会破坏教程可信度。 | 记录缺失模型和预期输出，模型到位后再补生成截图。 |
| 参数一次改太多 | 无法判断变化来自哪里。 | 固定其他参数，只改本章变量。 |
