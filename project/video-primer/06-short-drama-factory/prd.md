# Short Drama Factory 产品需求文档

## 1. 背景

AI 视频短剧制作不是单次视频生成任务，而是一个多阶段生产流程：创意需要扩写成剧本，剧本需要拆成镜头，镜头需要稳定的角色、场景和提示词，生成结果需要评分、返工、筛选和粗剪。

当前课程已经沉淀了 ComfyUI + Wan2.2 的手工流程，包括 T2V、I2V、FLF2V、角色一致性、分镜、批量筛选、后期和项目记录模板。`short-drama-factory` 要把这些流程自动化为一个可执行工具，但保留必要的人类确认点，避免全自动流程在剧本、连续性和质量上失控。

## 2. 产品目标

`short-drama-factory` 是一个 AI 短剧制作编排器。它不替代 ComfyUI，而是在 ComfyUI 之上完成创意到可剪视频素材的结构化编排。

MVP 的目标：

1. 用户输入一个短剧创意，工具自动生成 brief、剧本、角色卡、场景卡和风格卡。
2. 工具自动把剧本拆成 6-12 个镜头，并生成每个镜头的 Wan2.2 提示词。
3. 工具自动创建短剧项目目录、运行记录、评分表和 ComfyUI 运行计划。
4. 工具可生成 patched workflow JSON，供用户拖入 ComfyUI 手动验证。
5. 工具可在自动模式下调用 ComfyUI API 生成镜头，收集输出并记录结果。
6. 工具可对生成结果做基础评分、筛选和返工建议。
7. 工具可用 ffmpeg 生成粗剪版短剧。

## 3. 非目标

MVP 不做以下事情：

- 不训练视频模型。
- 不替代 ComfyUI 节点编辑器。
- 不保证无人值守生成最终商业级成片。
- 不内置复杂 Web 协作平台。
- 不承诺同 seed 跨 GPU、跨 ComfyUI 版本逐像素复现。
- 不自动使用未授权图片、音乐、字体、商标或人物肖像素材。

## 4. 用户角色

| 角色 | 需求 |
| --- | --- |
| 独立创作者 | 从一句创意快速得到可执行短剧项目包和分镜提示词。 |
| AI 视频课程学习者 | 按课程 Wan2.2 流程生成、记录、评分和返工短剧镜头。 |
| 小团队制片人 | 让不同 AI agent 分工生成剧本、分镜、提示词和 QC 报告。 |
| 工程维护者 | 通过 Shanforge 管理需求、任务、记忆、演进和质量门禁。 |

## 5. MVP 用户流程

```text
输入创意
-> 生成 brief
-> 生成剧本和润色版
-> 生成角色/场景/风格 bible
-> 拆分镜头
-> 生成镜头提示词
-> 生成素材需求和 ComfyUI 运行计划
-> 人工确认
-> 生成 patched workflow 或调用 ComfyUI API
-> 收集输出
-> 评分筛选
-> 返工或粗剪
```

## 6. 功能需求

### REQ-001 项目初始化

作为创作者，我希望输入项目名和创意后自动创建标准项目目录，以便后续生成、记录和交付都能归档。

优先级：P0

验收标准：

- AC-1：给定项目名 `rainy-night-001`，当执行初始化命令，则创建 `brief/`、`script/`、`bible/`、`shots/`、`inputs/`、`workflows/`、`outputs/`、`logs/`、`memory/` 目录。
- AC-2：给定已存在项目目录，当重复初始化，则不覆盖已有用户文件，并返回冲突提示。
- AC-3：初始化后必须生成 `project.json`，记录项目 ID、创建时间、目标时长、默认工作流后端和当前阶段。

### REQ-002 创意转 Brief

作为创作者，我希望工具把一句创意扩展为结构化 brief，以便后续 agent 有共同事实源。

优先级：P0

验收标准：

- AC-1：给定一句创意，当执行 `generate-brief`，则输出题材、目标时长、主角、冲突、风格、交付规格和限制。
- AC-2：brief 必须写入 `brief.md` 和 `brief.json`。
- AC-3：brief 生成后必须等待人工确认或显式传入 `--auto-approve`。

### REQ-003 剧本生成与润色

作为创作者，我希望工具生成短剧大纲、剧本和润色版，以便把创意变成可拍内容。

优先级：P0

验收标准：

- AC-1：给定已确认 brief，当执行 `generate-script`，则生成 `outline.md`、`script_v01.md` 和 `script_v02_polished.md`。
- AC-2：剧本必须包含场次、动作、对白、情绪节奏和预计时长。
- AC-3：剧本润色不能改变 brief 中的硬性限制，例如目标时长、角色数量和风格边界。

### REQ-004 一致性资料生成

作为创作者，我希望自动生成角色、场景和风格 bible，以便所有镜头保持一致。

优先级：P0

验收标准：

- AC-1：给定剧本，当执行 `generate-bible`，则生成 `characters.json`、`scenes.json`、`style.json`。
- AC-2：角色资料必须包含可复用身份字段、服装字段、情绪弧线和参考图需求。
- AC-3：场景资料必须包含空间关系、光线、色彩、时间和连续性限制。

### REQ-005 剧本拆分镜头

作为制片人，我希望工具把剧本拆成镜头清单，以便逐镜头生成而不是一次生成整片。

优先级：P0

验收标准：

- AC-1：给定剧本和 bible，当执行 `plan-shots`，则生成 6-12 个镜头。
- AC-2：每个镜头必须包含镜头目的、剧情信息、景别、机位、动作、时长、输入素材、推荐工作流、失败条件、剪辑点和音频点。
- AC-3：总时长与 brief 目标时长偏差不得超过 20%，否则返回 warning。

### REQ-006 提示词生成与优化

作为创作者，我希望自动生成每个镜头的正向和负向提示词，以便直接用于 Wan2.2 工作流。

优先级：P0

验收标准：

- AC-1：每个镜头必须生成 `identity block`、`scene block`、`action block`、`camera block`、`style block` 和 `negative prompt`。
- AC-2：角色镜头必须复用 `characters.json` 中的固定身份字段。
- AC-3：相邻镜头必须显式记录运动方向、色彩连续性和剪辑点。

### REQ-007 素材需求规划

作为创作者，我希望工具自动列出参考图、首帧和尾帧需求，以便提前准备 I2V/FLF2V 输入。

优先级：P0

验收标准：

- AC-1：每个 I2V 镜头必须指定参考图需求。
- AC-2：每个 FLF2V 镜头必须指定首帧、尾帧和构图一致性要求。
- AC-3：缺少必要素材时，生成阶段必须阻止执行并提示缺失项。

### REQ-008 ComfyUI 运行计划

作为创作者，我希望工具生成 ComfyUI 运行计划，以便知道每个镜头使用哪个工作流和参数。

优先级：P0

验收标准：

- AC-1：每个镜头必须映射到 T2V、I2V、TI2V 或 FLF2V 中的一类工作流。
- AC-2：运行计划必须包含 workflow JSON、prompt、seed、steps、width、height、frames、输入图片和输出命名。
- AC-3：运行计划必须写入 `shots/generation-plan.json`。

### REQ-009 Patched Workflow 生成

作为学习者，我希望工具生成已经填好参数的 ComfyUI workflow JSON，以便拖进 ComfyUI 检查和手动运行。

优先级：P0

验收标准：

- AC-1：给定某个镜头的运行计划，当执行 `export-workflow`，则生成对应 patched workflow JSON。
- AC-2：patched workflow 必须替换 prompt、negative prompt、Load Image、seed、尺寸、帧数和输出前缀。
- AC-3：如果无法定位工作流节点，则返回错误并说明缺失节点类型或节点 ID。

### REQ-010 ComfyUI 自动生成

作为进阶用户，我希望工具能调用 ComfyUI API 自动生成镜头，以便减少重复手工操作。

优先级：P1

验收标准：

- AC-1：给定运行中的 ComfyUI 服务，当执行 `generate-shot`，则提交队列并返回 job ID。
- AC-2：工具必须记录队列状态、耗时、输出文件和错误信息。
- AC-3：API 失败时不得静默跳过，必须写入 `logs/runs.jsonl`。

### REQ-011 结果评分和筛选

作为制片人，我希望工具对生成结果进行评分，以便区分保留、重跑和淘汰。

优先级：P1

验收标准：

- AC-1：评分维度至少包含主体识别、运动稳定、构图可剪、风格一致和技术瑕疵。
- AC-2：评分结果必须写入 `logs/scores.jsonl` 和 `outputs/selects/`、`outputs/rejects/` 的整理建议。
- AC-3：命中一票否决项时，结论必须是重跑或淘汰。

### REQ-012 返工建议

作为创作者，我希望工具能根据评分自动给出返工建议，以便减少盲目重跑。

优先级：P1

验收标准：

- AC-1：给定评分低于阈值的镜头，工具必须指出失败类别：角色、场景、运动、构图、显存、模型、素材或节点。
- AC-2：返工建议必须包含修改哪个字段，而不是泛泛要求“优化提示词”。
- AC-3：返工版本必须生成新 prompt 版本号，不覆盖旧版本。

### REQ-013 粗剪导出

作为创作者，我希望工具把 A 级候选镜头粗剪成短剧样片，以便快速检查叙事节奏。

优先级：P1

验收标准：

- AC-1：给定每个镜头至少一个 A 级候选，当执行 `rough-cut`，则输出 `outputs/edit/v01_rough.mp4`。
- AC-2：粗剪必须按 shot-list 顺序拼接，并记录每个片段的来源文件。
- AC-3：缺少关键镜头时，工具必须阻止导出或标记为不完整粗剪。

### REQ-014 记忆沉淀

作为维护者，我希望每次项目结束后沉淀成功参数和失败模式，以便后续项目复用。

优先级：P1

验收标准：

- AC-1：项目状态必须写入 `memory/current-state.md`。
- AC-2：成功 prompt、失败模式和可复用参数必须写入 `memory/evolution-notes.md`。
- AC-3：跨项目共享记忆必须脱敏，不包含未授权素材路径和私人内容。

## 7. 非功能需求

| ID | 类型 | 目标 |
| --- | --- | --- |
| NFR-001 | 可复现性 | 每次生成必须记录 prompt 版本、seed、workflow、模型后端、输入素材、输出文件和耗时。 |
| NFR-002 | 可审计性 | 所有 AI 生成的剧本、分镜、提示词和评分必须落盘，不只保存在会话上下文。 |
| NFR-003 | 安全性 | 默认不上传本地素材到未知第三方服务；外部 LLM 调用必须显式配置 provider。 |
| NFR-004 | 可扩展性 | LLM、ComfyUI 和视频编辑后端必须通过适配器接口接入。 |
| NFR-005 | 可靠性 | ComfyUI API 失败、节点缺失、模型缺失、显存不足必须返回明确错误类别。 |
| NFR-006 | 性能 | 非生成类 CLI 命令在普通项目规模下 P95 小于 5 秒，不含 LLM 调用时间。 |
| NFR-007 | 成本控制 | 默认每个镜头只生成 1 个 seed；批量 seed 必须显式开启。 |
| NFR-008 | 兼容性 | MVP 支持 macOS、Windows、Linux 上的本地 ComfyUI HTTP API。 |

## 8. 成功指标

| 指标 | MVP 目标 |
| --- | ---: |
| 从创意到完整项目包 | 小于 10 分钟，不含 LLM 排队时间 |
| 6 镜头 shot-list 完整率 | 100% |
| Prompt 必填字段完整率 | 100% |
| ComfyUI patched workflow 可打开率 | 大于 90% |
| 自动生成运行日志覆盖率 | 100% |
| 粗剪导出成功率 | 大于 80%，前提是关键镜头存在 |

## 9. 依赖

- 本地或远程 ComfyUI 服务。
- Wan2.2 工作流 JSON。
- 至少一个文本 LLM provider。
- 可选视觉模型 provider，用于自动评分。
- ffmpeg，用于粗剪和格式处理。
- Shanforge，可选但推荐，用于工程治理、skills、memory 和演进。

## 10. 风险

| 风险 | 影响 | 缓解 |
| --- | --- | --- |
| 多 agent 输出互相矛盾 | 剧本、分镜、提示词不一致 | 以 brief、bible 和 shot-list 作为唯一事实源。 |
| ComfyUI 节点版本变化 | patched workflow 无法运行 | workflow patcher 记录节点定位失败，并保留手工模式。 |
| 角色一致性不稳定 | 短剧观感断裂 | 强制角色 bible、参考图、I2V 和连续性评分。 |
| 全自动重跑成本过高 | 时间和显存浪费 | 默认半自动，批量 seed 显式开启。 |
| 版权和素材授权不清 | 交付风险 | 素材来源和授权状态进入项目记录。 |
