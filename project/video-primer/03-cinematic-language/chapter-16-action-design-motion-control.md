## 显存档位建议

| 显存 | 推荐做法 | 风险控制 |
| ---: | --- | --- |
| 8GB | 只做 动作控制 的低分辨率草稿：优先 5B、短帧数、单 seed。 | 不同时跑多个候选，不加载 14B 双阶段大模型。 |
| 12GB | 可完成 5B 完整练习；14B 只做小尺寸验证或等待 fp8/量化版本。 | 每次只改一个变量，失败先降帧数。 |
| 16GB | 可做 14B 小中尺寸流程，并保留草稿和精修两套参数。 | 先筛 seed，再提升分辨率。 |
| 24GB | 可做 动作控制 的标准练习和 2-4 个候选对比。 | 仍要记录 seed、模型、steps、宽高、帧数、耗时。 |

## 知识点：低风险动作

低风险动作幅度小，主体结构变化少，适合第一轮测试。

### 实例：眨眼微笑转头

| 项目 | 内容 |
| --- | --- |
| 输入 | 人像参考图；提示词：`the person blinks, smiles gently, slight head turn`。 |
| 操作 | 使用 I2V，动作只写小幅面部变化。 |
| 预期现象 | 脸部运动轻微，身份更容易保持。 |
| 判断原则 | 这是人物 I2V 的入门动作组合。 |

操作流程：

1. 打开本章对应的 Wan2.2 工作流或项目记录表。
2. 在记录表里写下变量：`动作=小幅人物动作`。
3. 输入本例提示词、素材或镜头要求。
4. 按显存档位选择草稿参数；本机缺模型时记录缺失文件，不伪造输出。
5. 截图保存节点、参数或项目表，并写下本例是否可进入下一步。

## 知识点：动作与镜头组合

主体动和镜头动不要一开始同时复杂。一个动，另一个就保持简单。

### 实例：主体静止镜头移动

| 项目 | 内容 |
| --- | --- |
| 输入 | `product stays still, slow dolly in, light sweep`。 |
| 操作 | 主体不动，只让镜头推进。 |
| 预期现象 | 画面有运动但产品稳定。 |
| 判断原则 | 这是产品广告常用安全组合。 |

1. 打开本章对应的 Wan2.2 工作流或项目记录表。

## 常见错误与排查

| 错误 | 常见原因 | 处理 |
| --- | --- | --- |
| 只写概念没有变量 | 不知道本章到底要验证什么。 | 每个实例只设置一个核心变量，并写入记录表。 |
| 结果失败但没有记录 | 缺少 seed、模型、提示词或截图。 | 先补记录，再决定是否重跑。 |
| 本机缺模型却写“已生成” | 伪造输出会破坏教程可信度。 | 记录缺失模型和预期输出，模型到位后再补生成截图。 |
| 参数一次改太多 | 无法判断变化来自哪里。 | 固定其他参数，只改本章变量。 |
