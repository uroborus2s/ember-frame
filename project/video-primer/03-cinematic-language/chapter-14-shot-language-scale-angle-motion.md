## 知识点：景别

景别决定观众看到的是人物情绪、主体动作还是环境关系。近景适合情绪，远景适合空间。

### 实例：近景人物情绪

| 项目 | 内容 |
| --- | --- |
| 输入 | `close-up of a tired young woman looking at rain on a window, subtle breathing, shallow depth of field` |
| 操作 | 在提示词中加入 `close-up` 和 `shallow depth of field`，保持动作很小。 |
| 预期现象 | 人物面部占画面主体，背景被弱化。 |
| 判断原则 | 近景的任务是表达情绪，不适合同时展示完整环境。 |

操作流程：

1. 打开本章对应的 Wan2.2 工作流或项目记录表。
2. 在记录表里写下变量：`景别=近景`。
3. 输入本例提示词、素材或镜头要求。
4. 按显存档位选择草稿参数；本机缺模型时记录缺失文件，不伪造输出。
5. 截图保存节点、参数或项目表，并写下本例是否可进入下一步。

## 知识点：机位

机位会改变观众对主体的心理感受。低机位显得强大，俯拍显得弱小或孤独。

### 实例：低机位英雄感

| 项目 | 内容 |
| --- | --- |
| 输入 | `low angle shot of a runner standing under dramatic light, heroic posture, slow camera push` |
| 操作 | 加入 `low angle shot`，不要同时加入俯拍词。 |
| 预期现象 | 主体显得更高、更有力量。 |
| 判断原则 | 低机位强化主体气势。 |

操作流程：

1. 打开本章对应的 Wan2.2 工作流或项目记录表。

4. 按显存档位选择草稿参数；本机缺模型时记录缺失文件，不伪造输出。
5. 截图保存节点、参数或项目表，并写下本例是否可进入下一步。

## 知识点：镜头运动

镜头运动会增加动态，但也增加失败风险。第一次练习优先慢推、慢摇，不要直接复杂运镜。

### 实例：dolly in 推进

| 项目 | 内容 |
| --- | --- |
| 输入 | `slow dolly in toward a glowing product on a black table, reflections, studio light` |
| 操作 | 加入 `slow dolly in`，主体保持静止。 |
| 预期现象 | 镜头逐渐靠近主体。 |
| 判断原则 | 推镜适合强调重点和高级感。 |

操作流程：

1. 打开本章对应的 Wan2.2 工作流或项目记录表。
2. 在记录表里写下变量：`镜头运动=慢推`。
3. 输入本例提示词、素材或镜头要求。
4. 按显存档位选择草稿参数；本机缺模型时记录缺失文件，不伪造输出。
5. 截图保存节点、参数或项目表，并写下本例是否可进入下一步。

## 常见错误与排查

| 错误 | 常见原因 | 处理 |
| --- | --- | --- |
| 只写概念没有变量 | 不知道本章到底要验证什么。 | 每个实例只设置一个核心变量，并写入记录表。 |
| 结果失败但没有记录 | 缺少 seed、模型、提示词或截图。 | 先补记录，再决定是否重跑。 |
| 本机缺模型却写“已生成” | 伪造输出会破坏教程可信度。 | 记录缺失模型和预期输出，模型到位后再补生成截图。 |
| 参数一次改太多 | 无法判断变化来自哪里。 | 固定其他参数，只改本章变量。 |
